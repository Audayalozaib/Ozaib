# توثيق مشروع تطبيق إدارة المستخدمين (PHP PDO)

يعتبر هذا المشروع نموذجاً متكاملاً لتطبيق ويب يعتمد على لغة **PHP** وتقنية **PDO** لإدارة عمليات تسجيل المستخدمين وتأمين بياناتهم. تم تصميم التطبيق ليكون متوافقاً مع بيئات النشر السحابية الحديثة مثل **Railway**، مع الالتزام بأفضل الممارسات الأمنية في التعامل مع قواعد البيانات وتشفير كلمات المرور.

## نظرة عامة على التقنيات المستخدمة

يعتمد التطبيق على مجموعة من التقنيات الأساسية التي تضمن الأداء العالي والأمان المطلوب في تطبيقات الويب الحديثة. يوضح الجدول التالي المكونات الرئيسية للمشروع:

| المكون | التقنية المستخدمة | الغرض منها |
| :--- | :--- | :--- |
| **قاعدة البيانات** | MySQL | تخزين بيانات المستخدمين بشكل دائم ومنظم. |
| **محرك الاتصال** | PDO (PHP Data Objects) | توفير طبقة تجريد للوصول لقاعدة البيانات وحمايتها من هجمات SQL Injection. |
| **تأمين البيانات** | Password Hashing (Bcrypt) | تحويل كلمات المرور إلى نصوص مشفرة غير قابلة للاسترجاع لضمان خصوصية المستخدمين. |
| **إدارة الجلسات** | PHP Sessions | تتبع حالة تسجيل دخول المستخدم وحماية الصفحات الخاصة. |

## تعليمات النشر والإعداد على Railway

لضمان تشغيل التطبيق بنجاح على منصة **Railway**، يجب اتباع خطوات محددة تبدأ بتهيئة قاعدة البيانات وتنتهي بربط متغيرات البيئة. يقوم التطبيق تلقائياً بالبحث عن متغيرات البيئة التي توفرها Railway للاتصال بقاعدة البيانات.

> **ملاحظة هامة:** يجب تنفيذ الاستعلامات الموجودة في ملف `schema.sql` داخل واجهة إدارة قاعدة البيانات في Railway قبل البدء باستخدام التطبيق لإنشاء الجداول اللازمة.

| متغير البيئة | الوصف |
| :--- | :--- |
| `MYSQLHOST` | عنوان مضيف قاعدة البيانات. |
| `MYSQLDATABASE` | اسم قاعدة البيانات المستخدمة. |
| `MYSQLUSER` | اسم المستخدم للوصول للقاعدة. |
| `MYSQLPASSWORD` | كلمة المرور الخاصة بقاعدة البيانات. |
| `MYSQLPORT` | المنفذ المستخدم للاتصال (غالباً 3306). |

## هيكل الملفات والوظائف

يتكون المشروع من مجموعة من الملفات المنظمة التي يؤدي كل منها وظيفة محددة في دورة حياة التطبيق. تبدأ الرحلة من صفحة الفهرس التي توجه المستخدم إلى بوابة تسجيل الدخول، ومن ثم يتم التحقق من هويته للوصول إلى المحتوى المحمي في الصفحة الرئيسية.

يحتوي ملف `db.php` على منطق الاتصال المركزي، بينما تتولى ملفات `register.php` و `login.php` معالجة مدخلات المستخدم والتحقق من صحتها. تم فصل التنسيقات الجمالية في ملف `style.css` لضمان سهولة التعديل والتطوير المستقبلي.
